# 從基準文本到正文定位

Status: Stable

此文檔解釋從基準文本開始，到生成基準文本樹，到最後爲基準文本定位的整個過程，及其中的一些細節。

## 生成原始文本（raw text）

- 在網上找可用的文本，如 <a href="https://github.com/JasonWade001/chtxt">JasonWade001/chtxt</a>
- 以 <a href="https://github.com/wingmingchan64/CanonicalTextTrees/blob/main/tools/php/bin/生成原始文本.php">生成原始文本.php</a> 去除格式、去除標點符號、規範用字
- 例子： <a href="https://github.com/wingmingchan64/CanonicalTextTrees/blob/main/論語/raw_text/論語.txt">論語.txt</a>

## 手編基準文本

- 按一定的格式，編輯原始文本，以編成基準文本
	- 篇目在第一行
	- 第二行爲空行
	- 段與段、章與章、闕（如宋詞）與闕之間，加入一行空行
	- 段/章/闕中可以適當分行
- 如果時間容許，以可靠版本原文，校對基準文本
- 例子： <a href="https://github.com/wingmingchan64/CanonicalTextTrees/blob/main/論語/canonical_text/01.txt">01.txt</a>

- 字路徑集記錄的是某書中所有單字、與單字相對應的路徑集

## 路徑集的用途

- 路徑集：決定在某書的基準正文樹中，某路徑是否合法、可循

>`function 是合法路徑( string $work_id, string $path ) : bool`

- 字路徑集：提取某書中某單字的所有路徑（該單字出現在甚麼地方）
- 路徑字集：以某字路徑（有字碼的路徑）提取該路徑指向的單字
- 句路徑集：提取某書中某單句的所有路徑（該單句出現在甚麼地方）

## 如何理解路徑

一個路徑，像 `LUNYU,15,42,85,11,3`，一般有六個部分：

- `LUNYU`：書名代號（代表《論語》），見<a href="https://github.com/wingmingchan64/CanonicalTextTrees/blob/main/schemas/json/registry/registry.json">registry.json</a>
- `15`：《論語》第十五卷 <a href="https://github.com/wingmingchan64/CanonicalTextTrees/blob/main/論語/canonical_text/15.txt">15.txt</a>
- `42`：卷中的第四十二章
- `85`：在正文文檔中的行碼，卽此正文片段在 `15.txt` 中的第 85 行
- `11`：第十一句（句的定義：在兩個句號之間的連續文字——第一句是例外）
- `3`：句中的第三個字
